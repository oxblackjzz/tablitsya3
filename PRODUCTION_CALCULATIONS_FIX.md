# Виправлення логіки розрахунків виробництва (ОНОВЛЕНО)

## ?? Проблема
Логіка розрахунків дат не відповідала вимогам:
- СТАРТ: мав бути обов'язково в робочі дні
- ФІНІШ: не враховувалося пропорційне перевищення потужності
- ВІДВАНТАЖЕННЯ: мав бути обов'язково робочий день

## ? Виправлення (ОСТАТОЧНА ВЕРСІЯ)

### 1. **СТАРТ виробництва**
**Було:** 
```csharp
var earliestProductionStartDate = orderDate.AddDays(daysBeforeProduction);
// Могло потрапити на вихідний
```

**Стало:**
```csharp
var earliestProductionStartDate = _workingDaysService.AddWorkingDays(orderDate, daysBeforeProduction);
// daysBeforeProduction = 2 РОБОЧІ дні
```

**Результат:** ? СТАРТ = Дата замовлення + 2 **РОБОЧІ** дні

---

### 2. **ФІНІШ виробництва**
**Було:** 
```csharp
// Просто +1 день при будь-якому перевищенні
if (hasCapacityOverflow)
{
    calculatedFinish = calculatedFinish.AddDays(1);
}
```

**Стало:**
```csharp
// Пропорційний розрахунок додаткових днів
double totalCapacityOverflow = 0; // Сумарне перевищення

// При розподілі виробництва рахуємо перевищення:
var overflow = productionDateLoad[currentDate] - dailyCapacity;
if (overflow > 0)
{
    totalCapacityOverflow += overflow;
}

// Розраховуємо додаткові дні пропорційно:
if (totalCapacityOverflow > 0)
{
    // Формула: (перевищення / потужність) округлено вгору
    var additionalDays = (int)Math.Ceiling(totalCapacityOverflow / dailyCapacity);
    calculatedFinish = calculatedFinish.AddDays(additionalDays);
}
```

**Результат:** 
- ? ФІНІШ = Кінець розподілу + 5 робочих днів
- ? ФІНІШ = ФІНІШ + **пропорційна кількість днів** при перевищенні потужності
- Приклад: перевищення 1500 м? при потужності 1000 м?/день ? +2 дні

---

### 3. **ВІДВАНТАЖЕННЯ**
**Було:** 
```csharp
var completionDate = productionEndDate.AddDays(1);
// Могло потрапити на вихідний
```

**Стало:**
```csharp
var completionDate = _workingDaysService.GetNextWorkingDay(productionEndDate);
// Наступний РОБОЧИЙ день
```

**Результат:** ? ВІДВАНТАЖЕННЯ = наступний **РОБОЧИЙ** день після фінішу

## ? Результат

Тепер розрахунки **повністю** відповідають вимогам:

1. ? **СТАРТ** розраховується динамічно, але не менше ніж +2 **РОБОЧІ** дні від дати замовлення
2. ? **ФІНІШ** розраховується: +5 робочих днів + **пропорційна кількість днів** при перевищенні потужності
3. ? **ВІДВАНТАЖЕННЯ** = наступний **РОБОЧИЙ** день після фінішу

## ?? Формула розрахунку додаткових днів

```csharp
// Збираємо сумарне перевищення потужності за всі дні
double totalCapacityOverflow = 0;

foreach (день виробництва)
{
    var overflow = завантаженість[день] - потужність;
    if (overflow > 0)
    {
        totalCapacityOverflow += overflow;
 }
}

// Розраховуємо додаткові дні (округлюємо вгору)
var additionalDays = ?totalCapacityOverflow / потужність?;

// Приклади:
// 500 м? / 1000 м?/день = 0.5 ? ?0.5? = 1 день
// 1500 м? / 1000 м?/день = 1.5 ? ?1.5? = 2 дні
// 2500 м? / 1000 м?/день = 2.5 ? ?2.5? = 3 дні
```

## ?? Математика перевищення

Формула округлення вгору (`Math.Ceiling`) забезпечує:
- 0.1-1.0 день перевищення ? +1 день до фінішу
- 1.1-2.0 дні перевищення ? +2 дні до фінішу
- 2.1-3.0 дні перевищення ? +3 дні до фінішу
- і т.д.

Це відповідає логіці: якщо потужність перевищена хоч трохи, потрібен мінімум 1 додатковий день.
