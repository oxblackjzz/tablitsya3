# ?? Виправлення пропорційного розрахунку СТАРТ (v3)

## ? Проблема в v2

У версії v2 була **критична помилка**:

```csharp
// 1. Рахували перевантаження ДО базового старту
var accumulatedOverflow = 0;
foreach (var kvp in productionDateLoad.Where(x => x.Key < earliestPossibleStart))
{
    var overflow = kvp.Value - dailyCapacity;
    if (overflow > 0) accumulatedOverflow += overflow;
}

// 2. Відкладали старт
var delayDays = (int)Math.Ceiling(accumulatedOverflow / dailyCapacity);
actualProductionStart = AddWorkingDays(earliestPossibleStart, delayDays);

// 3. Але потім розподіл створював НОВЕ перевантаження!
productionDateLoad[currentDate] += toAllocate;
```

### Результат:
- **Перше замовлення:** немає перевантаження ? старт +1 день ?
- **Друге замовлення:** рахує перевантаження, але розподіл накладається ? **неправильний старт** ?
- **Третє замовлення:** попереднє перевантаження не враховано ? **старт раніше ніж треба** ?

## ? Рішення в v3

### Нова логіка:

```csharp
// Замість розрахунку перевантаження - ШУКАЄМО ПЕРШИЙ ВІЛЬНИЙ ДЕНЬ
DateTime actualProductionStart = FindNextAvailableStartDate(
    earliestPossibleStart, 
    orderSize, 
    dailyCapacity, 
    productionDateLoad,
    workshopNumber);
```

### Метод `FindNextAvailableStartDate`:

```csharp
private DateTime FindNextAvailableStartDate(
    DateTime earliestStart, 
    double orderSize, 
    int dailyCapacity, 
  Dictionary<DateTime, double> productionDateLoad,
    int workshopNumber)
{
    var candidateDate = earliestStart;
    var remainingToAllocate = orderSize;

    // Симулюємо розподіл замовлення
    while (remainingToAllocate > 0)
    {
        // Пропускаємо неробочі дні
        while (!IsWorkingDay(candidateDate))
        {
      candidateDate = candidateDate.AddDays(1);
     }

        // Перевіряємо завантаженість на цю дату
   var currentLoad = productionDateLoad.ContainsKey(candidateDate) 
      ? productionDateLoad[candidateDate] 
        : 0;

      var availableCapacity = dailyCapacity - currentLoad;

        if (availableCapacity > 0)
     {
            // ЦЕХ МАЄ ВІЛЬНУ ПОТУЖНІСТЬ - ЦЕ НАШ СТАРТ!
            return candidateDate;
        }

        // Цех повністю завантажений - пробуємо наступний день
        candidateDate = candidateDate.AddDays(1);
    }

    return candidateDate;
}
```

## ?? Переваги v3

### ? Що виправлено:

1. **Точний розрахунок старту** - враховує реальну завантаженість цеху
2. **Немає дублювання** - не рахуємо перевантаження двічі
3. **Динамічний пошук** - шукаємо перший день з вільною потужністю
4. **Правильна черга** - замовлення не накладаються понад потужність

### ? Що залишилося:

- **ФІНІШ:** +5 робочих днів + пропорційне перевищення (без змін)
- **ВІДВАНТАЖЕННЯ:** наступний робочий день (без змін)

## ?? Технічні деталі

### Метод `CountWorkingDaysBetween`:

```csharp
public int CountWorkingDaysBetween(DateTime startDate, DateTime endDate)
{
    if (endDate <= startDate)
        return 0;

    var workingDays = 0;
    var currentDate = startDate.AddDays(1); // Не включаємо початкову дату

    while (currentDate <= endDate)
    {
   if (IsWorkingDay(currentDate))
            workingDays++;
  currentDate = currentDate.AddDays(1);
    }

    return workingDays;
}
```

Використовується для логування затримки у **робочих днях**.
