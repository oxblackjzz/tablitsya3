# 📋 Документація системи "Планування Виробництва" (Tablitsya3)

## 🎯 Загальний опис

**Tablitsya3** — це веб-застосунок на базі Blazor Server для планування та управління виробництвом меблів. Система забезпечує повний цикл від прийому замовлень до контролю виготовлення деталей.

---

## 📦 Основні модулі та функції

### 1. 🏠 **Головна панель (Dashboard)**
**Сторінка:** `/`

| Функція | Опис |
|---------|------|
| KPI статистика | Відображення загальної кількості замовлень, активних цехів, обсягу в роботі |
| Завантаженість цехів | Візуалізація завантаження кожного цеху у відсотках |

---

### 2. 📊 **Планування виробництва**
**Сторінки:** `/production-planning`, `/gantt-combined`

| Функція | Опис |
|---------|------|
| Діаграми Ганта | Візуалізація графіку виробництва по цехам |
| Об'єднана діаграма | Зведений вигляд всіх цехів на одній діаграмі |
| Розрахунок потужності | Автоматичний розрахунок завантаження цеху (м²/день) |
| Розподіл замовлень | Алгоритм оптимального розподілу замовлень по днях |
| Робочі дні | Врахування вихідних та святкових днів |

---

### 3. 📝 **Управління замовленнями**
**Сторінки:** `/orders`, `/bulk-order-entry`

| Функція | Опис |
|---------|------|
| Перегляд замовлень | Таблиця всіх замовлень з фільтрацією |
| Масове введення | Швидке додавання кількох замовлень |
| Редагування | Зміна параметрів замовлень (обсяг, дати, цех) |
| Видалення | Видалення замовлень з підтвердженням |

---

### 4. 📂 **Імпорт проектів**
**Сторінка:** `/scanning/import`

| Функція | Опис |
|---------|------|
| Парсинг XML | Імпорт файлів `.project` з програми розкрою |
| Валідація | Перевірка структури XML файлу |
| Дедублікація | Пропуск вже існуючих деталей |
| Перезапис | Можливість очистити існуючі дані перед імпортом |

**Дані що імпортуються:**
- Проекти (UUID, вартість, кількість деталей)
- Товари (назва, код, опис, кількість)
- Деталі (розміри, матеріал, кромка, QR-код)

---

### 5. 📷 **Сканування деталей**
**Сторінка:** `/scanning`

| Функція | Опис |
|---------|------|
| Сканування QR | Фіксація проходження деталі через етап |
| Етапи виробництва | Розкрій → Кромка → Свердління → Сортування |
| Прив'язка працівника | Ідентифікація хто виконав операцію |
| Прив'язка станції | Фіксація на якому обладнанні виконано |
| Історія сканувань | Журнал всіх операцій з деталлю |

**Етапи виробництва:**
```
0 - Невизначено
1 - Розкрій (Cut)
2 - Кромкування (EdgeBanding)
3 - Свердління (Drilling)
4 - Сортування/Пакування (Sorting)
```

---

### 6. 👷 **Управління працівниками**
**Сторінки:** `/workers`, `/workers/statistics`

| Функція | Опис |
|---------|------|
| Реєстрація | Створення працівника з кодом та PIN |
| Авторизація | Вхід за кодом/PIN або бейджем |
| Статистика | Кількість відсканованих деталей |
| Продуктивність | Графіки ефективності по періодах |
| Дозволені етапи | Обмеження доступу до певних операцій |

---

### 7. 🖥️ **Управління станціями**
**Сторінка:** `/workstations`

| Функція | Опис |
|---------|------|
| Реєстрація | Додавання нової станції/станка |
| Прив'язка пристрою | Автоматичне визначення за DeviceID |
| Статус | Активна/Неактивна |
| Етап | До якого етапу виробництва належить |

---

### 8. ⚠️ **Управління браком**
**Сторінка:** `/defects`

| Функція | Опис |
|---------|------|
| Реєстрація браку | Фіксація дефектної деталі |
| Типи дефектів | Скол, тріщина, подряпина, неправильний розмір тощо |
| Статуси | Новий → В роботі → Виправлено/Списано |
| Перевиробництво | Друк нової бірки для заміни |

**Типи дефектів:**
- Скол
- Тріщина
- Подряпина
- Неправильний розмір
- Дефект матеріалу
- Дефект кромки
- Дефект свердління
- Забруднення
- Деформація
- Інше

---

### 9. 🏭 **Налаштування цехів**
**Сторінка:** `/workshop-settings`

| Функція | Опис |
|---------|------|
| Додавання цеху | Створення нового цеху з назвою та номером |
| Потужність | Налаштування денної продуктивності (м²) |
| Робочі години | Графік роботи цеху |
| Кольори | Візуальне оформлення на діаграмах |

---

### 10. 💾 **Резервне копіювання**
**Сторінка:** `/backup-management`

| Функція | Опис |
|---------|------|
| Створення бекапу | Ручне збереження поточного стану |
| Автобекап | Автоматичне збереження перед змінами |
| Відновлення | Повернення до попереднього стану |
| Історія | Список останніх 10 бекапів |

---

### 11. 📄 **Журнал подій**
**Сторінка:** `/logs`

| Функція | Опис |
|---------|------|
| Перегляд логів | Всі системні події |
| Фільтрація | За типом, датою, модулем |
| Рівні | Info, Warning, Error, Debug |

---

### 12. 🏷️ **Друк бірок**
**Компонент:** `PrintDefectLabel.razor`

| Функція | Опис |
|---------|------|
| Генерація бірки | HTML для друку з QR-кодом |
| QR-код | Унікальний ідентифікатор деталі |
| Штрих-код | Code128 для сканера |
| Позначка браку | Спеціальна мітка для перевиробництва |

## 🗄️ Структура даних

### Основні сутності:

```
ImportedProjectEntity   - Імпортований проект
├── ProductEntity       - Товар/Виріб
│   └── PartEntity      - Деталь
│
WorkerEntity            - Працівник
├── WorkerSessionEntity - Сесія працівника
│
WorkstationEntity       - Робоча станція
│
ScanLogEntity           - Журнал сканувань
│
DefectEntity            - Брак/Дефект
```

## 🔧 Технічний стек

| Компонент | Технологія |
|-----------|------------|
| Backend | ASP.NET Core 8.0, Blazor Server |
| Database | SQLite + Entity Framework Core |
| UI | Bootstrap 5, Bootstrap Icons |
| QR-коди | QRCoder |
| Графіки | Custom CSS/SVG |

## 📁 Структура проекту

```
Tablitsya3/
├── Components/
│   ├── Layout/           # Макет сторінок
│   ├── Pages/            # Сторінки (Razor)
│   └── Shared/           # Спільні компоненти
├── Data/
│   ├── Entities/         # Entity класи
│   └── ApplicationDbContext.cs
├── Models/               # DTO та моделі
├── Services/             # Бізнес-логіка
├── wwwroot/              # Статичні файли
│   └── js/app.js         # JavaScript
└── Program.cs            # Точка входу
```

## 🚀 Швидкий старт

1. **Налаштування цехів** → `/workshop-settings`
2. **Додавання працівників** → `/workers`
3. **Додавання станцій** → `/workstations`
4. **Імпорт проекту** → `/scanning/import`
5. **Планування** → `/production-planning`
6. **Сканування** → `/scanning`
