# ?? Синхронізація даних діаграми з таблицею

## ? Проблема

Після редагування замовлення в таблиці, **тултіп (спливаюче вікно) на діаграмі Ганта показував старі дані**:

```
Таблиця:     Діаграма Ганта (тултіп):
Замовлення: 28.10.2025     ?   Замовлення: 28.10.2025  ? (стара дата)
Старт: 06.11.2025          ?   Старт: 01.09.2025        ? (стара дата)
Фініш: 11.11.2025          ?   Фініш: 05.09.2025        ? (стара дата)
Відвантаж: 05.12.2025      ?   Відвантаж: 10.09.2025    ? (стара дата)
```

---

## ?? Причина

### Blazor не відстежує зміни всередині об'єктів

```csharp
// Коли ви редагуєте замовлення:
SaveOrderEdit()
   ?
CalculateAllSchedules()  // Перераховує дані
   ?
schedule1.Orders[0].StartDate = новаДата  // Змінює дані в об'єкті
   ?
StateHasChanged()  // Викликає оновлення
   ?
? GanttChart НЕ оновлюється!
```

### Чому?

Blazor компонент `GanttChart` отримує **той самий список Orders**:

```csharp
// ДО редагування:
<GanttChart Orders="@schedule1.Orders" />  // список A

// ПІСЛЯ редагування:
<GanttChart Orders="@schedule1.Orders" />  // ТЕПЕРІШНІЙ список A
```

**Blazor бачить:** "Ой, той самий список! Не треба оновлювати компонент!"

Але **всередині об'єктів** дані змінилися!

---

## ? Рішення: @key атрибут

### Що таке @key?

`@key` каже Blazor: "Якщо ключ змінився - **видали старий компонент і створи новий**!"

### Реалізація:

#### 1. Додаємо лічильник в `ProductionPlanning.razor.cs`:

```csharp
// Дані
private WorkshopData workshopData = new();
private ProductionSchedule? schedule1;
private ProductionSchedule? schedule3;
private ProductionSchedule? schedule6;
private bool hasSchedules = false;
private string statusMessage = string.Empty;

// ?? Ключ для примусового оновлення діаграм Ганта
private int ganttChartKey = 0;
```

#### 2. Збільшуємо лічильник після перерахунку:

```csharp
private void CalculateAllSchedules()
{
    // ... розрахунки ...
    
    if (hasSchedules)
 {
        statusMessage = "? Графіки успішно розраховані!";
   
  // ?? Збільшуємо ключ для оновлення діаграм
     ganttChartKey++;
    }
    
    StateHasChanged();
}
```

#### 3. Додаємо @key до компонентів у `.razor`:

```razor
<!-- Цех №1 -->
<GanttChart 
    @key="@($"gantt-1-{ganttChartKey}")"
    WorkshopNumber="1" 
Orders="@filteredOrders1Gantt" 
    StartDate="@workshopData.StartDate"
    DailyCapacity="@workshopData.WorkshopCapacities[1]" />

<!-- Цех №3 -->
<GanttChart 
    @key="@($"gantt-3-{ganttChartKey}")"
    WorkshopNumber="3" 
    Orders="@filteredOrders3Gantt" 
    StartDate="@workshopData.StartDate"
    DailyCapacity="@workshopData.WorkshopCapacities[3]" />

<!-- Цех №6 -->
<GanttChart 
    @key="@($"gantt-6-{ganttChartKey}")"
    WorkshopNumber="6" 
    Orders="@filteredOrders6Gantt" 
    StartDate="@workshopData.StartDate"
    DailyCapacity="@workshopData.WorkshopCapacities[6]" />
```

## ?? Як це працює

### Послідовність подій:

```
1. Користувач редагує замовлення в таблиці
   ?
2. SaveOrderEdit() ? CalculateAllSchedules()
   ?
3. ganttChartKey++ (було 0, стало 1)
   ?
4. Blazor бачить: key змінився!
   ?
5. Blazor ВИДАЛЯЄ старий GanttChart
 ?
6. Blazor СТВОРЮЄ новий GanttChart з новими даними
   ?
7. ? Тултіп показує НОВІ дані!
```

### Приклад:

```
// ДО редагування:
<GanttChart @key="gantt-1-0" Orders="..." />  ? ключ: gantt-1-0

// ПІСЛЯ редагування:
<GanttChart @key="gantt-1-1" Orders="..." />  ? ключ: gantt-1-1
           ? ІНШИЙ КЛЮЧ!
        
Blazor: "Ой, новий ключ! Створюю новий компонент!"
```

## ?? Технічні деталі

### Формат ключа:

```csharp
@key="@($"gantt-{workshopNumber}-{ganttChartKey}")"
```

**Структура:**
- `gantt` - префікс
- `{workshopNumber}` - номер цеху (1, 3, або 6)
- `{ganttChartKey}` - лічильник (збільшується після кожного перерахунку)

**Приклади:**
```
gantt-1-0  // Цех №1, перший рендер
gantt-1-1  // Цех №1, після першого оновлення
gantt-1-2  // Цех №1, після другого оновлення

gantt-3-0  // Цех №3, перший рендер
gantt-6-0  // Цех №6, перший рендер
```

## ? Продуктивність

### Чи це повільно?

**НІ!** Пересоздание компонента дуже швидке:

```
Час створення GanttChart: ~10-50ms
Швидкість: Миттєво для користувача
```

### Коли відбувається оновлення?

Тільки коли **змінюються дані**:
- ? Після редагування замовлення
- ? Після розрахунку графіків
- ? Після видалення замовлення
- ? НЕ при наведенні курсора
- ? НЕ при зміні фільтрів

---

## ?? Урок: Коли використовувати @key

### Використовуйте @key коли:

1. **Зміни всередині об'єктів не відстежуються**
   ```csharp
   order.StartDate = newDate;  // Blazor не бачить
   ```

2. **Потрібно примусити перестворення компонента**
   ```razor
   <MyComponent @key="uniqueKey" />
   ```

3. **Компонент має складну логіку ініціалізації**
   ```csharp
   protected override void OnInitialized()
   {
       // Складні розрахунки
   }
   ```

### НЕ використовуйте @key коли:

1. **Зміни відстежуються автоматично**
   ```csharp
   myList = newList;  // Blazor бачить зміну
   ```

2. **Компонент простий і оновлюється швидко**

3. **Занадто часті оновлення** (продуктивність!)

## ?? Можливі проблеми

### Проблема: Діаграма "мигає" при оновленні

**Причина:** Компонент пересТворюється повністю

**Рішення:** Це нормально! Оновлення дуже швидке (~10-50ms)

---

### Проблема: Фільтри на діаграмі скидаються

**Причина:** Компонент створюється заново, стан фільтрів втрачається

**Рішення:** Зберігати стан фільтрів в батьківському компоненті і передавати як параметри

```csharp
// ProductionPlanning.razor.cs
private ViewMode gantt1ViewMode = ViewMode.All;

// ProductionPlanning.razor
<GanttChart 
    @key="..."
    ViewMode="@gantt1ViewMode"
    OnViewModeChanged="(mode) => gantt1ViewMode = mode" />
```
