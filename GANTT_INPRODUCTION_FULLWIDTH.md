## ?? Технічні деталі

### Логіка визначення ширини:

```csharp
int minWidthPx;

if (currentViewMode == ViewMode.Pending && allCalendarDates.Count <= 30)
{
    // Незавершені ?30 днів: розтягуємо на 100%
    minWidthPx = 100;
}
else if (currentViewMode == ViewMode.InProduction)
{
    // ?? В роботі: завжди розтягуємо на 100%
    minWidthPx = 100;
}
else
{
// Усі інші: фіксована ширина з прокруткою
    minWidthPx = allCalendarDates.Count * 80;
}

var ganttWidth = Math.Max(minWidthPx, 100);
```

### CSS стилі:

```razor
<div class="gantt-timeline" style="@(minWidthPx == 100 ? "width: 100%; min-width: 100%;" : $"width: {ganttWidth}px; min-width: {ganttWidth}px;")">
```

**Що відбувається:**

- `minWidthPx == 100` ? `width: 100%; min-width: 100%;` ? Розтягування
- `minWidthPx != 100` ? `width: 4800px; min-width: 4800px;` ? Фіксовано

## ?? Чому саме режим "В роботі"?

### Особливості режиму "В роботі":

1. **Мало замовлень** - тільки ті що виробляються зараз (4-8 замовлень)
2. **Короткий період** - зазвичай 7-14 днів
3. **Фокус на деталях** - користувач хоче бачити поточну роботу **детально**

### Порівняння з іншими режимами:

| Режим | Замовлень | Днів | Ширина |
|-------|-----------|------|--------|
| **Усі** | 25 | 60 | 4800px (прокрутка) ? |
| **Незавершені** | 15 | 25-35 | 100% або 2800px |
| **В роботі** | 4-8 | 7-14 | 100% (розтягнуто) ?? |
