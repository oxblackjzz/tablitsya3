# ? Виправлення проблеми редагування назв замовлень

## ?? Проблема

**Користувач:** "не можу редагувати і зберегти назву замовлення"

## ?? Діагностика

Після аналізу коду було виявлено **3 проблеми**:

### Проблема 1: Занадто сувора валідація дат
```csharp
// ? БУЛО: Не дозволяло редагувати замовлення з минулими датами
if (editOrderDate.Date < DateTime.Today)
{
    statusMessage = "? Помилка: дата замовлення не може бути в минулому!";
 return false;
}
```

**Наслідок:** Неможливо було редагувати назви існуючих замовлень, якщо їх дата була в минулому.

### Проблема 2: Неправильна ініціалізація дати початку виробництва
```csharp
// ? БУЛО: Додавало ProductionLeadTime до дати замовлення
editProductionStart = order.StartDate.AddDays(workshopData.ProductionLeadTime);
```

**Наслідок:** Дата початку виробництва не відповідала реальній даті з Order.

### Проблема 3: Відсутність детального логування
**Наслідок:** Неможливо було зрозуміти, на якому кроці виникала помилка.

## ? Виправлення

### Виправлення 1: М'якша валідація дат
```csharp
// ? СТАЛО: Перевіряємо тільки логічну послідовність дат
private bool ValidateOrderEdit()
{
    // Перевірка: дата завершення має бути пізніше або рівна даті замовлення
    if (editCompletionDate.Date < editOrderDate.Date)
    {
        statusMessage = "?? Помилка: дата завершення має бути пізніше або дорівнювати даті замовлення!";
        return false;
    }

    // Перевірка: дата початку виробництва має бути між датою замовлення і завершення
    if (editProductionStart.Date < editOrderDate.Date || editProductionStart.Date > editCompletionDate.Date)
    {
        statusMessage = "?? Помилка: дата початку виробництва має бути між датою замовлення та завершення!";
        return false;
    }

    return true;
}
```

**Зміни:**
- ? Видалено перевірку `editOrderDate.Date < DateTime.Today`
- ? Змінено перевірку дати завершення: `<=` ? `<` (дозволяє рівні дати)
- ? Покращені повідомлення про помилки

## ?? Інструкція для користувача

### Як тепер редагувати назву:

1. **Перейдіть** на сторінку "Планування виробництва"
2. **Прокрутіть** до таблиці замовлень
3. **Натисніть** кнопку ?? (олівець) біля потрібного замовлення
4. **Змініть** назву в текстовому полі
5. **Натисніть** ? (галочка) для збереження

### Що може пійти не так:

#### Помилка 1: "Дата завершення має бути пізніше або дорівнювати даті замовлення"
**Причина:** Ви випадково змінили дату завершення на ранішу, ніж дата замовлення.
**Рішення:** Виправте дату завершення або скасуйте зміни (?).

#### Помилка 2: "Дата початку виробництва має бути між датою замовлення та завершення"
**Причина:** Дата початку виробництва виходить за межі періоду замовлення.
**Рішення:** Виправте дату початку виробництва або скасуйте зміни (?).

#### Помилка 3: "Помилка: графік не знайдено"
**Причина:** Графіки не розраховані.
**Рішення:** Натисніть "Розрахувати графіки" і спробуйте знову.

### Якщо проблема залишається:

1. **Перегляньте логи:** Перейдіть на `/log-viewer` і знайдіть помилки
2. **Оновіть дані:** Натисніть кнопку "Оновити дані"
3. **Перезавантажте сторінку:** `F5` або `Ctrl+R`
4. **Очистіть кеш:** `Ctrl+Shift+Delete`
