# Діагностика проблеми зміни дат

## Проблема
Не можу змінити дату завершення або початку виробництва в режимі редагування замовлення.

## Внесені зміни для виправлення

### 1. Заміна InputDate на input type="date"
Замінено компонент `<InputDate>` на стандартний HTML `<input type="date">` для всіх трьох полів дат:
- Дата замовлення
- Дата початку виробництва  
- Дата завершення виробництва

### 2. Додано правильні прив'язки
```razor
<input type="date" 
       @bind="EditOrderDateValue"
       @bind:event="onchange" 
        
       class="form-control form-control-sm" />
```

### 3. Додано логування
У метод `StartOrderEdit` додано детальне логування для відстеження значень дат.

## Як перевірити

### Крок 1: Запустити додаток
```bash
cd таблиця3
dotnet run
```

### Крок 2: Перейти на сторінку планування
1. Відкрити браузер
2. Перейти до `/production-planning`
3. Натиснути "Розрахувати графіки" (якщо потрібно)

### Крок 3: Редагувати замовлення
1. Знайти будь-яке замовлення в таблиці
2. Натиснути кнопку "Редагувати" (іконка олівця)
3. Рядок має підсвітитися жовтим (`table-warning`)
4. Спробувати змінити дати в полях:
   - **Дата** - дата замовлення
- **Поч. вироб.** - дата початку виробництва
   - **Заверш.** - дата завершення виробництва

### Крок 4: Зберегти зміни
1. Натиснути зелену галочку (кнопка "Зберегти")
2. Дані мають збе регтися автоматично
3. Графіки мають перерахуватися

## Можливі причини проблеми

### 1. Браузер не підтримує input type="date"
**Рішення:** Використовувати сучасний браузер (Chrome, Edge, Firefox останні версії)

### 2. Дати не оновлюються в пам'яті
**Перевірка:** Відкрити консоль розробника (F12) та переглянути логи

### 3. Проблема з форматом дати
HTML `input type="date"` очікує формат `yyyy-MM-dd`, але Blazor може передавати інший формат.

**Рішення:** Додано `` до всіх інпутів

### 4. Проблема з InteractiveServer режимом
У файлі `ProductionPlanning.razor` є `@rendermode InteractiveServer`.

**Перевірка:** Переконатися що SignalR підключення активне (немає помилок у консолі браузера)

## Альтернативне рішення

Якщо `input type="date"` не працює, можна повернутися до використання текстових полів з маскою:

```razor
<input type="text" 
@bind="editOrderDateString" 
       @bind:event="onchange"
       class="form-control form-control-sm" 
       placeholder="дд.мм.рррр"
       pattern="\d{2}\.\d{2}\.\d{4}" />
```

І додати властивості для конвертації:

```csharp
private string editOrderDateString
{
    get => editOrderDate.ToString("dd.MM.yyyy");
    set
  {
     if (DateTime.TryParseExact(value, "dd.MM.yyyy", null, 
          System.Globalization.DateTimeStyles.None, out var date))
        {
            editOrderDate = date;
        }
    }
}
```

## Логи для перевірки

Система логує наступні події:
1. **Початок редагування:** `"Редагування замовлення №{X}..."` з усіма датами
2. **Збереження:** `"Дата завершення збережена: {дата}"`
3. **Перерахунок:** `"Перерахунок графіків..."`

Перевірити логи можна:
- У браузері (Console, F12)
- На сторінці `/log-viewer` (якщо така є)
- У файлах логів системи

## Що робити далі

### Якщо проблема залишається:

1. **Перевірте консоль браузера (F12)** на наявність JavaScript помилок
2. **Перевірте Network вкладку** - чи відправляються SignalR повідомлення
3. **Спробуйте інший браузер** - Chrome або Edge останньої версії
4. **Очистіть кеш браузера** - Ctrl+Shift+Delete
5. **Перезавантажте сервер** - зупиніть і знову запустіть `dotnet run`

### Для детальнішої діагностики:

Додайте `console.log` у JavaScript для перевірки значень:

```javascript
// wwwroot/js/app.js
document.addEventListener('change', function(e) {
    if (e.target.type === 'date') {
        console.log('Date changed:', e.target.value);
    }
});
```

## Контактна інформація

Якщо проблема не вирішується, надайте:
1. Скріншот сторінки з відкритим режимом редагування
2. Лог з консолі браузера (F12 ? Console)
3. Лог з сервера (вивід команди `dotnet run`)
