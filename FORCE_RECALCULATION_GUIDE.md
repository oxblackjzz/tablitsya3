# ?? Інструкція: Як оновити дані після зміни логіки (v2)

## ? Проблема
Після зміни логіки розрахунків дані в таблиці не оновилися, бо:
1. Старі дані завантажуються з файлу `workshop-data.json`
2. Кастомні дати завершення блокують перерахунок
3. Потрібен **примусовий перерахунок** з очищенням кастомних дат

---

## ?? Що змінилося в v2?

### СТАРТ виробництва:
```
v1: СТАРТ = замовлення + 2 робочі дні (фіксовано)
v2: СТАРТ = замовлення + 1 робочий день + пропорційна затримка при перевантаженні
```

**Формула v2:**
```
СТАРТ = orderDate + 1 робочий день
        + ?накопичене_перевищення / потужність? робочих днів
```

**Приклади:**
- Немає перевантаження ? +1 день
- Перевантаження 500 м? (потужність 1000) ? +1 + ?0.5? = +2 дні
- Перевантаження 2500 м? (потужність 1000) ? +1 + ?2.5? = +4 дні

---

## ? Рішення: Кнопка "Примусовий перерахунок"

На сторінці **"Планування виробництва"** є жовта кнопка:

```
?? Примусовий перерахунок
```

### Що вона робить:
1. ?? Видаляє **ВСІ** кастомні дати завершення
2. ?? Перераховує графіки **з нуля** за новою логікою v2
3. ?? Автоматично зберігає оновлені дані

---

## ?? Покрокова інструкція

### Крок 1: Перезапустити додаток
```
1. Зупиніть додаток (Shift+F5)
2. Запустіть знову (F5 або Ctrl+F5)
```

?? **Чому?** 
- Hot Reload не підтримує зміни параметрів за замовчуванням
- Нова логіка СТАРТ потребує повного перезапуску

---

### Крок 2: Відкрити "Планування виробництва"
```
1. Перейдіть на сторінку "Планування виробництва"
2. Дані завантажаться автоматично (зі старою логікою)
```

---

### Крок 3: Натиснути "Примусовий перерахунок"
```
1. Натисніть жовту кнопку "?? Примусовий перерахунок"
2. Підтвердіть дію в діалоговому вікні
3. Зачекайте кілька секунд
```

?? **Увага!** Це видалить всі ручні зміни дат завершення!

---

### Крок 4: Перевірити результати
```
1. Перегляньте таблицю внизу сторінки
2. Переконайтеся, що дати змінилися:
   ? СТАРТ - мінімум +1 робочий день від замовлення
   ? СТАРТ - пропорційно більше при перевантаженні
   ? ФІНІШ - з урахуванням перевищення потужності
   ? ВІДВАНТАЖЕННЯ - робочий день (наступний після фінішу)
```

## ?? Перевірка результатів v2

### Як перевірити нову логіку СТАРТ?

#### Приклад 1: Перше замовлення (немає перевантаження)
```
Дано:
- Замовлено: 14.10.2025 (вівторок)
- Площа: 1000 м?
- Попередні замовлення: немає

Очікуваний результат:
- СТАРТ: 15.10.2025 (середа) = 14.10 + 1 робочий день
- Немає затримки через перевантаження
```

#### Приклад 2: З перевантаженням попередніх
```
Дано:
- Замовлено: 15.10.2025 (середа)
- Площа: 1200 м?
- Попередні замовлення: перевищення 500 м?
- Потужність: 1000 м?/день

Очікуваний результат:
- Базовий старт: 15.10 + 1 = 16.10 (четвер)
- Накопичене перевищення: 500 м?
- Додаткова затримка: ?500/1000? = 1 день
- СТАРТ: 17.10.2025 (п'ятниця) = базовий + 1 день
```

#### Приклад 3: З великим перевантаженням
```
Дано:
- Замовлено: 16.10.2025 (четвер)
- Площа: 3000 м?
- Попередні замовлення: перевищення 2500 м?
- Потужність: 1000 м?/день

Очікуваний результат:
- Базовий старт: 16.10 + 1 = 17.10 (п'ятниця)
- Накопичене перевищення: 2500 м?
- Додаткова затримка: ?2500/1000? = 3 дні
- СТАРТ: 20.10.2025 (понеділок) = базовий + 3 робочі дні
```

## ?? Альтернативні методи

### Метод 1: Очистити всі дані
```
1. Натиснути "? Очистити все"
2. Додати замовлення знову
```

### Метод 2: Видалити файл вручну
```
1. Знайти файл: `таблиця3\Data\workshop-data.json`
2. Видалити файл
3. Перезапустити додаток
4. Додати замовлення знову
```

### Метод 3: Змінити налаштування
```
1. Перейти на сторінку "Налаштування цехів"
2. Змінити "Затримка старту" на 1 день
3. Натиснути "Зберегти"
4. Повернутися на "Планування" і натиснути "Примусовий перерахунок"
```

## ? FAQ v2

**Q: Чому після натискання "Розрахувати графіки" нічого не змінилося?**  
A: Кастомні дати блокують перерахунок. Використовуйте "Примусовий перерахунок".

**Q: Чи втрачу я ручні зміни дат?**  
A: Так. "Примусовий перерахунок" видаляє всі кастомні дати.

**Q: Як перевірити, що працює нова логіка СТАРТ?**  
A: Перегляньте логи - там буде текст "Відкладено старт на X днів через перевантаження".

**Q: Чому дані не оновлюються без перезапуску?**  
A: Параметри за замовчуванням (`DaysBeforeProduction = 1`) вимагають перезапуску.

**Q: Як працює нова формула СТАРТ?**  
A: `СТАРТ = замовлення + 1 день + ?перевищення/потужність?`. Чим більше перевантаження попередніх замовлень, тим пізніше старт.
